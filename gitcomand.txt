git config	утилита, которая позволяет просматривать и настраивать параметры Git.

Эти параметры могут быть сохранены в трёх местах:

Файл /etc/gitconfig содержит значения, общие для всех пользователей системы и для всех их репозиториев. Если при запуске git config указать параметр --system, то параметры будут читаться и сохраняться именно в этот файл.

Файл ~/.gitconfig или ~/.config/git/config хранит настройки конкретного пользователя. Этот файл используется при указании параметра --global.

Файл config в каталоге Git (т.е. .git/config) репозитория, который вы используете в данный момент, хранит настройки конкретного репозитория.

dir	 просмотр содиржимого в папке.

X: - сделать текущим диском диск X

cd /(каталог)		вход в папку

git config --list --show-origin		посмотр всех настроек и где они заданы.

git config --list	проверка конфигурации.


git help <команда>	 помощь при использовании Git.

git <команда> -h	вывод краткой инструкции по использованию.

git init	создаёт в текущей директории новую поддиректорию с именем .git, содержащую всю структуру Git-		репозитория.



git status	проверка состояния файла в системе GIT.

git status -s	 более упрощенный вывод.

 git status -s
 AM README	A - файл проиндексирован(подготовлен для коммита staged area, под версионным контролем);
		М - файл modified(модифицирован и неподготовлен for commit, данные могут пропасть).

.gitignore	игнорировать файлы и не добавлять в репозиторий.



git commit --amend		перезаписать коммит.
